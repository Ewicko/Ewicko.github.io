# Blog-First Site Redesign

**Date:** 2026-03-18
**Status:** Approved

## Overview

Redesign ethanwicko.com from a static HTML portfolio site into a blog-first personal site using Hugo. The blog is the primary attraction; the project portfolio becomes a secondary section. The visual direction is ultra-minimal, inspired by Dan Luu, Paul Graham, and Daring Fireball — serif typography, light background, pure content focus.

## Goals

1. Add a blog as the primary section of the site
2. Replace the "vibecoded" dark-mode aesthetic with a timeless, ultra-minimal design
3. Switch from hand-written HTML to Hugo so new posts are just Markdown files
4. Preserve all existing project content
5. Keep deployment on GitHub Pages with the ethanwicko.com custom domain

## Site Structure

```
/                     → Blog index (homepage)
/blog/:slug/          → Individual blog post
/projects/            → Project portfolio grid
/projects/:slug/      → Individual project page
/about/               → About page
/index.xml            → RSS feed (auto-generated by Hugo, blog posts only)
```

### Homepage (`/`)

The homepage is the blog index. It lists posts in reverse chronological order.

Each entry shows:
- Date (e.g., "March 2026")
- Title as a link to the post
- One-line description/summary

No images, no cards, no tags on the index. Just a clean list of writing.

Above the list: site name and navigation. No hero section, no tagline.

### Blog Post (`/blog/:slug/`)

Clean single-column reading layout:
- Title (large, Georgia serif)
- Date below title
- Body content in Georgia at ~17px, line-height ~1.7
- Max content width: 640px
- Support for: code blocks with syntax highlighting, LaTeX math via KaTeX, images, blockquotes, footnotes
- Share links at bottom of post (see Social Features section)
- Giscus comment section at bottom (see Social Features section)

### Projects Page (`/projects/`)

A grid of project cards, carrying over the spirit of the current portfolio:
- Card with thumbnail image (16:9 aspect ratio)
- Title and short description
- Technology tags as small pills
- Click to go to individual project page
- Responsive grid: auto-fit columns, min 260px
- Max width: 1100px

The 7th project ("Older Engineering Projects") renders as a card that links externally to the Notion portfolio archive. It does not get its own project page.

### Individual Project Page (`/projects/:slug/`)

Converted from existing HTML project pages. Max content width: 1100px (wider than blog posts to accommodate two-column layouts with images).

Each project page includes:
- Breadcrumb navigation (Home → Projects → Project Name)
- Project metadata (Type, Duration, Status)
- Overview section with text and images/video
- Technical details section
- Feature highlights
- Tech stack badges
- Previous/Next project navigation
- KaTeX math support via `math: true` front matter (same as blog posts)

### About Page (`/about/`)

Single page with:
- Who Ethan is and what he works on (sourced from the `#about` section of the current `index.html` — ignore the existing root `about.md` which contains unrelated quantum mechanics content)
- Links to email, LinkedIn, GitHub

### 404 Page

Custom `layouts/404.html` — simple, minimal:
- "Page not found" heading
- Link back to homepage
- Same nav and styling as the rest of the site

## Design Language

### Typography

- **Body text:** Georgia, 'Times New Roman', serif — 17px, line-height 1.7
- **Headings:** Georgia, serif — normal weight for h1/h2 on blog, can be bolder on project pages
- **Navigation and metadata:** -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif — 13-14px
- **Code:** Menlo, Monaco, 'Courier New', monospace — 14px

### Colors

Intentionally limited palette:

- **Background:** #fafafa (page), #ffffff (cards/code blocks)
- **Text:** #1a1a1a (headings), #333333 (body), #999999 (metadata/dates)
- **Links:** #0066cc, underlined on hover only
- **Code blocks:** #f5f5f5 background, #333 text
- **Borders:** #e0e0e0 (subtle dividers between posts)

No dark mode. One mode, done well.

### Layout

- Max content width: 640px for blog posts and about page (optimal reading line length)
- Max content width: 1100px for projects grid and individual project pages
- Centered content with generous vertical spacing
- Sticky header is not needed — the page is simple enough without one

### Navigation

Top of page, simple layout:
- Left: site name "ethanwicko.com" (plain text, links to home)
- Right: Blog · Projects · About
- Font: sans-serif, 13px
- Separated by a subtle bottom border from content
- Active page indicated by bold weight (e.g., "**Blog**" when on blog pages)

## Social Features

### Comments (Giscus)

- Uses [giscus](https://giscus.app) — powered by GitHub Discussions
- Loads at the bottom of each blog post, below share links
- Lazy-loaded: only loads the giscus iframe when the user scrolls to the comment section (via `loading="lazy"` on the iframe or Intersection Observer)
- Light theme to match the site aesthetic
- Configured in `hugo.toml` with repo, category, and mapping settings
- Rendered via a `partials/comments.html` template included in `blog/single.html`

### Share Links

Plain text links at the bottom of each blog post (above comments):
- "Share on X" — pre-filled tweet with post title and URL
- "Share on LinkedIn" — LinkedIn share URL
- "Copy link" — copies the post URL to clipboard (small JS snippet)
- Styled as simple inline text links, not buttons or icons. Matches the minimal aesthetic.
- Rendered via a `partials/share.html` template

### Subscribe

- RSS feed prominently linked in the site footer and on the about page: "Subscribe via RSS"
- RSS output limited to blog posts only (configured via `[outputs]` in `hugo.toml`)
- No email newsletter for now — RSS is the subscription mechanism

### Hugo Configuration (`hugo.toml`)

Key settings:
- `baseURL = "https://ethanwicko.com/"`
- Blog section as the homepage (set `content/blog/` as main sections)
- Chroma syntax highlighting (style: monokailight or similar light theme)
- RSS feed enabled, limited to blog posts: `[outputs] home = ["HTML", "RSS"]` with `[outputFormats.RSS]` scoped to blog section
- Clean URL structure with trailing slashes
- Permalinks: `[permalinks] blog = "/blog/:slug/"`, `projects = "/projects/:slug/"`

### Head Partial (`partials/head.html`)

The `<head>` element includes:
- `<meta charset="utf-8">`
- `<meta name="viewport" content="width=device-width, initial-scale=1">`
- `<meta name="robots" content="noai, noimageai">` (carried over from current site)
- `<meta name="theme-color" content="#fafafa">` (updated from dark theme)
- `<title>` — page title, falling back to site title
- `<meta name="description">` — page description from front matter, falling back to site description
- Open Graph tags: `og:title`, `og:description`, `og:url`, `og:type` (article for posts, website for others), `og:image` (from post/project front matter `thumbnail` field; no fallback homepage OG image — omit if not set)
- `twitter:card` — summary_large_image when og:image is set, summary otherwise
- `<link rel="canonical">` — canonical URL for the page
- `<link rel="alternate" type="application/rss+xml">` — RSS feed link
- `<link rel="icon" type="image/png" href="/favicon.png">`
- `<link rel="icon" type="image/x-icon" href="/favicon.ico">` (fallback)
- `<link rel="stylesheet" href="/css/style.css">`
- Conditional KaTeX CSS/JS (only when `math: true` in front matter)

### GitHub Actions Deployment

A `.github/workflows/deploy.yml` that:
1. Checks out the repo
2. Installs Hugo (extended version for potential SCSS later)
3. Runs `hugo --minify`
4. Deploys `public/` to GitHub Pages

The `.nojekyll` file is no longer needed (Hugo generates its own output). The CNAME file moves to `static/CNAME` so Hugo copies it to the build output.

### KaTeX Integration

- Load KaTeX CSS and JS only on pages that use math (conditional in template)
- Use Hugo's front matter flag: `math: true` to enable per-post and per-project
- Render `$...$` for inline and `$$...$$` for display math

### Syntax Highlighting

- Hugo's built-in Chroma highlighter (no client-side JS needed)
- Light theme to match the minimal aesthetic
- Fenced code blocks with language annotation: `` ```python ``, `` ```go ``, etc.

### Video Embedding

The current project pages include `.mp4` and `.gif` files. A custom Hugo shortcode `layouts/shortcodes/video.html` handles video embedding in Markdown:

```
{{</* video src="/images/FlexureDiff/episode_animation.mp4" */>}}
```

This renders a `<video>` tag with controls, autoplay muted, and responsive width. GIFs can be embedded as regular Markdown images.

Each existing HTML project page becomes a Markdown file in `content/projects/`:

1. `compliant-mechanism-generative.html` → `compliant-mechanism-generative.md`

Front matter for each project:
```yaml
---
title: "FlexureDiff"
description: "Diffusion model for flexure generation"
type: "projects"
thumbnail: "/images/FlexureDiff/cover.png"
tags: ["PyTorch", "DDPM", "Generative Design"]
duration: "2024 - Present"
status: "Active"
weight: 1  # controls ordering on projects page
math: false  # set to true for projects with equations (e.g., FlexureDiff)
---
```

### About Content

Sourced from the `#about` section of the current `index.html` homepage. The existing root `about.md` (quantum mechanics content) is unrelated and should be removed. Contact links (email, LinkedIn, GitHub) included.

### Images

All image directories stay as-is, moved to `static/images/`. No renaming or restructuring needed.

### Assets to Preserve

- `favicon.ico` → `static/favicon.ico`
- `favicon.png` → `static/favicon.png` (optimize from 1.3MB to <50KB before migration)

## What's Intentionally Excluded

- **No dark mode** — one mode, done well, avoids "vibecoded" aesthetic
- **No tag/category index pages** — can add later if the post count warrants it
- **No email newsletter** — RSS is the subscription mechanism for now
- **No analytics** — privacy-respecting by default
- **No third-party theme** — custom templates for full control

## Success Criteria

1. Writing a new blog post is: create a `.md` file in `content/blog/`, push to `main`, done
2. The site loads fast with no layout shift and minimal JavaScript (giscus lazy-loaded, KaTeX conditional, clipboard copy)
3. Blog posts render cleanly with code blocks, math, images, share links, and comments
4. All existing project content is preserved and accessible
5. The site looks timeless and professional — someone visiting in 5 years won't think it's dated
6. Deploys automatically on push to `main` via GitHub Actions
